# fix(release): prevent duplicate crates.io publishing

**PR #177** | CLOSED | Author: charliecreates[bot] | Created: 2026-07-15 | Merged: 2026-07-15

Branch: `fix/con-1465-single-crates-publisher` → `main`

## Summary

- Remove the duplicate tag-triggered `auto-publish-crates-io` job from `release.yml`.
- Keep `release-strict.yml` as the sole automatic crates.io publisher for `v*.*.*` tags.
- Preserve both workflows' manual `workflow_dispatch` publish paths.
- Do not bump `2.0.11`; the published release remains unchanged.

The observed `v2.0.11` race had `Release` run [29387278011](https://github.com/Conxian/conxius-enclave-sdk/actions/runs/29387278011) publish successfully, followed by `Release Strict` run [29387278023](https://github.com/Conxian/conxius-enclave-sdk/actions/runs/29387278023) failing because `2.0.11` already existed on crates.io. Removing the duplicate tag-triggered job fixes the systemic race without creating a new version.
