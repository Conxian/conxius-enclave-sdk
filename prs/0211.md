# fix(wasm): close key and provider boundary

**PR #211** | CLOSED | Author: charliecreates[bot] | Created: 2026-07-20 | Merged: 2026-07-20

Branch: `fix/200-wasm-key-boundary` → `main`

## Summary

- Removes the WASM `WasmArkClient::derive_vutxo_key` export that serialized private key bytes as hex.
- Adds provider-backed public-key/signing capability names while keeping unapproved providers and all documented WASM runtime lanes fail-closed.
- Removes WASM seed-based Ark recovery and secret-bearing Fedimint serialization; makes BitVM2 construction fallible instead of creating a localhost/software default.
- Adds negative boundary tests, a runtime/provider support matrix, and migration guidance.

## API change and migration

This is an unreleased breaking change for the `2.x` line. See [`docs/migrations/wasm-key-boundary.md`](https://github.com/Conxian/conxius-enclave-sdk/blob/d6a86f721af6f5830eecea4a4847a855fea0443f/docs/migrations/wasm-key-boundary.md) for:

- removal of `derive_vutxo_key(seedHex, index)`;
- provider-owned `derive_vutxo_public_key` and `sign_vutxo` capability names;
- typed `UNSUPPORTED_RUNTIME`, `UNSUPPORTED_PROVIDER`, and `SECRET_EXPORT_FORBIDDEN` errors;
- fallible BitVM2 construction and the `taproot_internal_key_hex` rename.

## Supported / unsupported matrix

[`docs/architecture/WASM_SUPPORT_MATRIX.md`](https://github.com/Conxian/conxius-enclave-sdk/blob/d6a86f721af6f5830eecea4a4847a855fea0443f/docs/architecture/WASM_SUPPORT_MATRIX.md) explicitly keeps Browser/Web, Node.js, Bundler, and Web Worker lanes **Unsupported**. Only Rust/WASM compilation is verified here; no browser, Node, bundler, worker, provider, hardware, or release-artifact acceptance claim is made.

Cloud, localhost, software-only, and mock implementations remain test/development paths and cannot satisfy production support evidence.
