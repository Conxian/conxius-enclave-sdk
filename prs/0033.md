# fix(enclave): remove fixed CloudEnclave fallback key

**PR #33** | CLOSED | Author: charliecreates[bot] | Created: 2026-04-16 | Merged: 2026-04-18

Branch: `charlie/fix-cloud-enclave-dev-key` → `master`

## Summary

Remove the hard-coded CloudEnclave fallback key from the enclave layer to eliminate an unsafe default path and ensure all enclave operations rely on properly provisioned keys.

## Changes

- Remove the fixed/embedded CloudEnclave fallback key from `src/enclave/cloud.rs`.
- Enforce use of provisioned/attested keys only; no implicit fallback path remains.
- Adjust key-loading logic to fail fast when configuration is missing or invalid instead of silently using a default key.
- Update associated error handling to surface meaningful errors to the caller when key material cannot be loaded.
- Refine/align tests with the new behavior so that:
  - Tests no longer rely on or construct the removed fallback key path.
  - Negative cases assert on explicit key-loading failures.

## Rationale

A fixed fallback key in the enclave compromises security guarantees and can:
- Introduce a shared secret across environments.
- Weaken isolation and trust assumptions of the TEE.
- Mask deployment/configuration issues by silently succeeding with an unsafe default.

This change ensures that enclave operations only proceed when legitimate key material is present and properly configured.

## Impact

- **Breaking behavior:** Environments that implicitly relied on the fallback key must now supply valid CloudEnclave key material (or equivalent configuration) for enclave operations to succeed.
- **Security:** Strengthens enclave security posture by removing an unsafe default secret.

## Testing

- Unit tests updated and passing for `src/enclave/cloud.rs` key-loading behavior.
- Verified that:
  - Correctly configured environments continue to function as expected.
  - Misconfigured environments now fail explicitly with clear error messages.

## Follow-ups

- Validate that deployment and CI pipelines always provision the required keys.
- Update any remaining documentation or examples that might have implied the existence of a fallback key.
