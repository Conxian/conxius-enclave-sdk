# fix(settlement): tighten ISO 20022 namespace checks

**PR #37** | CLOSED | Author: charliecreates[bot] | Created: 2026-04-16 | Merged: 2026-04-18

Branch: `charlie/iso20022-namespace-scope-helper` → `master`

Follow-up to the PR #36 review thread: dedupe namespace parsing and tighten ISO 20022 namespace validation so the `FIToFICstmrCdtTrf` trigger can’t be satisfied by lookalike / malformed xmlns values.

### Changes

- Refactor `validate_iso20022_trigger_payload()` to centralize `NamespaceScope` construction for `Event::Start` and `Event::Empty`.
- Tighten ISO 20022 namespace validation:
  - Require the `urn:iso:std:iso:20022:tech:xsd:` prefix.
  - Validate the message identifier shape (e.g. `pacs.008.001.08`) to reject suffix junk.
- Add regression tests for:
  - `urn:iso:std:iso:20022evil...` (prefix lookalike)
  - malformed message IDs (`...pacs.008.001.08evil`)
