# fix: fail closed on simulated threshold protocols

**PR #209** | OPEN | Author: charliecreates[bot] | Created: 2026-07-20 | Merged: never

Branch: `fix/197-disable-simulated-protocols` → `main`

## Summary

This draft quarantines production-facing simulated behavior from issue #197 without inventing replacement cryptography.

- Adds stable typed `ConclaveError::ProtocolUnsupported` status with protocol, operation, and reason fields.
- Removes simulated hashes, fixture discovery, structural proof acceptance, local settlement transitions, placeholder IDs, and synthetic signatures from the target APIs.
- Keeps public data/serialization types where they remain safe, but value-bearing operations now fail before returning simulated material.
- Updates the affected WASM adapters to propagate the typed unsupported failures.
- Replaces positive simulation tests with explicit negative tests that verify no local state mutation and exact unsupported status.

## Disabled paths

- `src/protocol/frost.rs`
  - key package generation
  - DKG round 1/2 generation and verification
  - signature-share aggregation
- `src/protocol/nexus/fedimint.rs`
  - federation registration/join
  - mint intent preparation and e-cash issuance
  - note verification
  - DLEQ generation/verification and blind request construction
  - threshold signature aggregation/validation
- `src/protocol/ark.rs`
  - V-UTXO key derivation
  - ASP recovery scan
  - vTXO tree construction
  - forfeit signing
- `src/protocol/bitvm2.rs`
  - forfeit construction
  - commitment posting
  - challenge submission and resolution
  - challenge status/window evaluation
  - forfeit signing

Pure data types and serialization boundaries remain available for future audited implementations; no simulated backend or alternate crypto implementation was added.

## Compatibility impact

This is intentionally fail-closed and includes API changes:

- `ArkManager::derive_vutxo_key` now returns `ConclaveResult<[u8; 32]>` instead of raw key bytes.
- `DleqProof::verify`, `FedimintAdapter::verify_note`, and `validate_threshold_signature` now return `ConclaveResult<bool>` so unsupported status is explicit instead of a boolean success/failure.
- Existing callers of FROST, Fedimint, Ark, and BitVM2 value-bearing paths must handle `ProtocolUnsupported` until audited implementations are supplied.
- Existing serialized data structs remain defined, but they cannot be produced by the disabled operations.
