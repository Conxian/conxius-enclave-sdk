Updates the requirements on [musig2](https://github.com/conduition/musig2) to permit the latest version.
<details>
<summary>Changelog</summary>
<p><em>Sourced from <a href="https://github.com/conduition/musig2/blob/master/CHANGELOG.md">musig2's changelog</a>.</em></p>
<blockquote>
<h2>v0.4.0</h2>
<ul>
<li><strong>DX</strong>: New benchmarks for verification, signing and comparison against libsecp256k1.</li>
<li><strong>SecNonce changes:</strong> To fully comply with BIP-327, the <code>SecNonce</code> and <code>SecNonceBuilder</code> API have been changed (<a href="https://redirect.github.com/conduition/musig2/pull/17">conduition/musig2#17</a>). The <code>SecNonce</code> data structure now includes the signer's public key. See below for migration instructions.</li>
<li><strong>Security Improvement:</strong> To align with BIP-327 security recommendations, we now validate the pubkey contained in <code>SecNonce</code> against the key provided by the signer in the <code>sign_partial</code> and <code>adaptor::sign_partial</code> functions, and if they do not align we return a new error enum member <code>SigningError::SecNoncePubkeyMismatch</code>. In the state-machine API, this can only happen if you pass in the secret key of a <em>different</em> pubkey within the same <code>KeyAggContext</code> as passed to <code>FirstRound::new</code>.</li>
</ul>
<h3>Breaking Changes</h3>
<ol>
<li>The <code>SecNonce</code> serialization format has been updated to comply with BIP327.</li>
</ol>
<pre lang="diff"><code> /// The size of a serialized [`SecNonce`] in bytes.
-pub const SEC_NONCE_SIZE: usize = 64;
+pub const SEC_NONCE_SIZE: usize = 97;
</code></pre>
<p>This affects any downstream callers who store <code>SecNonce</code> out of band, and load it later (e.g long-lived signing sessions).</p>
<p><strong>Recommended Fix:</strong> Add middleware code or perform database migration to append the signer's individual 33-byte public key before deserializing a <code>SecNonce</code> created by <code>musig2</code> version v0.3.1 or earlier.</p>
<ol start="2">
<li>The <code>SecNonceBuilder</code> API has changed.</li>
</ol>
<pre lang="diff"><code> impl&lt;'snb&gt; SecNonceBuilder&lt;'snb&gt; {
-    pub fn new(nonce_seed: impl Into&lt;NonceSeed&gt;) -&gt; SecNonceBuilder&lt;'snb&gt;;
-    pub fn with_pubkey(self, pubkey: impl Into&lt;secp::Point&gt;) -&gt; SecNonceBuilder&lt;'snb&gt;;
-    pub fn with_seckey(self, seckey: impl Into&lt;secp::Scalar&gt;) -&gt; SecNonceBuilder&lt;'snb&gt;;
+    pub fn from_pubkey(
+        nonce_seed: impl Into&lt;NonceSeed&gt;,
+        pubkey: impl Into&lt;secp::Point&gt;,
+    ) -&gt; SecNonceBuilder&lt;'snb&gt;;
+    pub fn from_seckey(
+        nonce_seed: impl Into&lt;NonceSeed&gt;,
+        seckey: impl Into&lt;secp::Scalar&gt;,
+    ) -&gt; SecNonceBuilder&lt;'snb&gt;;
 }
</code></pre>
<p>To comply with BIP-327, a public or secret key is now a mandatory parameter needed to build a <code>SecNonce</code>. As such, the <code>SecNonceBuilder::with_pubkey</code> and <code>SecNonceBuilder::with_seckey</code> methods have been removed, and the <code>SecNonceBuilder::new(nonce_seed)</code> constructor method has been replaced with two new constructors:</p>
<ul>
<li><code>SecNonceBuilder::from_pubkey(nonce_seed, pubkey)</code>, which takes any type that converts to <code>secp::Point</code>.</li>
<li><code>SecNonceBuilder::from_seckey(nonce_seed, seckey)</code>, which takes any type that converts to <code>secp::Scalar</code>.</li>
</ul>
<p><strong>Recommended Fix:</strong> The state-machine API will handle this adjustment for you. If you use the state-machine API, no action should be needed. If you use the functional API, you will need to update your code to provide a public key, or if possible a secret key, when building a <code>SecNonce</code> using one of the two new constructors. Callers who wish to update a <code>SecNonceBuilder</code> with a secret key after calling a constructor may still do so, using the <code>SecNonceBuilder::with_spices</code> method, but beware this overwrites any public key (or secret key) set by the constructor.</p>
<ol start="3">
<li>The <code>SecNonce</code> API has changed.</li>
</ol>
<pre lang="diff"><code> impl SecNonce {
-    pub fn new&lt;T: Into&lt;secp::Scalar&gt;&gt;(k1: T, k2: T) -&gt; SecNonce;
&lt;/tr&gt;&lt;/table&gt;
</code></pre>
