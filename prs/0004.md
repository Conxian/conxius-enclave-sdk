# Enhance SDK Security and Align CLP with CON-15 Audit

**PR #4** | CLOSED | Author: botshelomokoka | Created: 2026-03-04 | Merged: 2026-03-04

Branch: `jules/sdk-security-and-clp-alignment-v2-650173268178953905` → `master`

This PR enhances the lib-conclave-sdk security posture and aligns the Stacks Concentrated Liquidity Pool (CLP) contract with the CON-15 audit findings.

Key changes:
1. **CLP Contract Improvements**: Introduced slippage protection (`min-amount-out`), tick range validation (`tick-lower < tick-upper`), and non-zero amount assertions in the `mint` and `swap` functions.
2. **Enclave Security**: Increased PBKDF2 iterations from 100k to 600k for session key derivation and enforced a minimum PIN length of 4 characters.
3. **Attestation Verification**: Added a simulated Root of Trust check to the `DeviceIntegrityReport::verify` method to ensure reports originate from a trusted Conclave Root CA.
4. **Protocol Refinements**: Added input validation to `RailProxy` and `StacksManager`. Added an `expiration` field to `AffiliateProof` to prevent replay attacks of referral proofs.
5. **Testing**: Expanded the test suite in `src/lib.rs` to cover new security checks, including invalid message hash sizes, short PINs, and attestation verification.
