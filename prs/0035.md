# fix(enclave): bound simulated keygen

**PR #35** | CLOSED | Author: charliecreates[bot] | Created: 2026-04-16 | Merged: 2026-04-18

Branch: `charlie/cloud-enclave-key-validation` → `master`

## Description
Follow-up to #29 / #34: `CloudEnclave::new` is now fallible and simulated KMS key generation is bounded (1024 attempts) to avoid the theoretical unbounded-loop hang when sampling invalid secp256k1 scalars.

This also updates the WASM constructor (`ConclaveWasmClient::new`) to propagate the construction failure as a `JsValue` rather than panicking.
