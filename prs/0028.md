# fix(enclave): remove fixed CloudEnclave fallback key

**PR #28** | CLOSED | Author: charliecreates[bot] | Created: 2026-04-16 | Merged: 2026-04-16

Branch: `charlie/fix-cloud-enclave-dev-key` → `master`

This is a follow-up to the PR #27 review thread: it removes the remaining fixed deterministic signing key behavior in `CloudEnclave` and makes the dev-key path non-panicking + zeroizable.

### Changes
- `CloudEnclave` now uses a per-instance simulated key when no dev key is configured (no more `[0xCD; 32]` fallback).
- `CloudEnclave::with_dev_key()` is now fallible (`ConclaveResult<Self>`) and stores dev key bytes in `Zeroizing<[u8; 32]>`.
