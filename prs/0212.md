# fix: fail closed CCTP, account abstraction, and asset adapters

**PR #212** | OPEN | Author: charliecreates[bot] | Created: 2026-07-20 | Merged: never

Branch: `fix/198-fail-closed-adapters-assets` → `main`

## Summary

- Add typed `ConclaveError::InvalidConfiguration` handling for malformed configuration and address/asset metadata.
- Validate EVM EIP-55 addresses plus supported Solana and Tron encodings; keep unverified asset records catalogued but inactive/quarantined.
- Restrict active built-in contract metadata to reviewed canonical records and reject non-mainnet asset use until network-specific evidence exists.
- Enforce asset validation before opportunity and rail discovery, intent preparation, broadcast, settlement proposal creation, and stablecoin balance reads.
- Make CCTP and ERC-7579 paths validate local input shape but return typed `Unsupported` results instead of producing placeholder calldata, accepting unbound attestations, or claiming executable module setup.
- Replace issue-scope placeholder fixtures and propagate typed adapter failures through WASM bindings.

This PR intentionally does **not** claim production support, release acceptance, or independent security review.

## Non-claims and remaining gates

- CCTP calldata encoding and Iris attestation verification remain disabled pending canonical Circle ABI/message vectors, signature verification, network integration, and review evidence.
- ERC-7579 execution and module setup remain disabled pending network-bound account, entry-point, module registry, interface, and code-provenance verification.
- Active asset scope is intentionally narrow; unreviewed regional, testnet/devnet, unsupported-chain, and placeholder records remain unavailable for value-bearing routing.
- This does not close the canonical Bitcoin/Ethereum, protocol-placeholder, toolchain/reproducibility, WASM/platform, or release-acceptance gates tracked by the related issues.
- The previous placeholder-returning adapter APIs now return typed results; callers must handle the intentional fail-closed behavior.
