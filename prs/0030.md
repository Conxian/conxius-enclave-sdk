# fix(wasm): avoid f64 for job card amount

**PR #30** | CLOSED | Author: charliecreates[bot] | Created: 2026-04-16 | Merged: 2026-04-16

Branch: `fix/wasm-job-card-amount` → `master`

Follow-up to the PR #27 review thread: remove floating point money from the WASM boundary by switching `generate_job_card` to take a decimal string amount.

## Description
- `generate_job_card(sender, receiver, amount_sbtc, ...)` now takes `amount_sbtc: String` (decimal string), not `f64`.
- `ConxianJobCard.work_intent.amount_sBTC` is now a string, validated as a simple positive decimal with up to 8 fractional digits.

Example JS call shape:
```ts
client.generate_job_card(
  sender,
  receiver,
  "0.05", // string (not number)
  "Johannesburg",
  "ZA",
)
```

Fixes the safety issue raised in https://github.com/Conxian/lib-conclave-sdk/pull/27#discussion_r3090323094.
