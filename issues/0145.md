# Enforce strict CI/CD baseline for enclave SDK build, verification, and release

**Issue #145** | CLOSED | Created: 2026-07-03 | Updated: 2026-07-08

Apply the strict CI/CD baseline to `conxius-enclave-sdk` as a critical SDK with security-sensitive implications.

Mandatory controls:
- pinned actions only
- required lint / test / build checks
- dependency review
- secret scanning
- static analysis and cargo audit where applicable
- release gating on verified artifacts
- SBOM generation for release artifacts
- provenance / attestation for release artifacts
- environment-gated production release path

Gap focus:
- strengthen release discipline around an SDK that can influence downstream security posture
- ensure workflow and action drift do not undermine trust claims

Acceptance criteria:
- enclave SDK cannot merge or release without the defined strict baseline
- release outputs are verifiable and suitable for downstream security-sensitive consumption
