# [P1] Define telemetry privacy, monitoring, and public-safe operational runbooks

**Issue #201** | OPEN | Created: 2026-07-20 | Updated: 2026-07-20

The audit identifies telemetry privacy, monitoring, and operational-runbook gaps. Without safe defaults, minimized payloads, and reliable release/incident evidence, production operation could either expose sensitive data or lack the controls needed to detect and recover from failures.

Refs #191

## Scope
- Define and test telemetry behavior in `src/telemetry.rs` and its integration with the rails.
- Specify opt-in or minimal-default behavior per surface, payload minimization, retention expectations, and failure semantics.
- Ensure telemetry never carries API credentials, private keys, raw attestation material, or equivalent sensitive values.
- Document TLS, endpoint, timeout, retry, and telemetry-failure behavior without publishing private operational details.
- Add public-safe monitoring, rollback, release-recovery, and ownership runbooks.

## Evidence
- [Audit PR #193](https://github.com/Conxian/conxius-enclave-sdk/pull/193) records telemetry/privacy and operational-runbook gaps as P1 findings.
- Relevant surfaces include `src/telemetry.rs`, telemetry integration in the rails, and readiness/security documentation.

## Acceptance criteria
- Telemetry defaults are explicit, tested, and consistent with the documented opt-in or minimal-default privacy posture for each surface.
- Payloads are minimized and tested to exclude API credentials, private keys, raw attestation evidence, and other sensitive values.
- TLS, endpoint validation, timeouts, retries, and telemetry-failure behavior are specified and covered by tests without making core security decisions fail open.
- Monitoring, rollback, release-recovery, and incident-escalation runbooks are public-safe, actionable, and assigned to named roles or owners.
- Documentation does not expose private endpoints, privileged identifiers, credentials, custody procedures, or recovery secrets.

## Dependencies
- Uses the implementation and evidence boundaries from the preceding child issues and must be included in the final acceptance review.
- Operational claims remain conditional until the exact release candidate has the required monitoring and recovery evidence.
