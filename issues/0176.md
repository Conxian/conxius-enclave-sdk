# Fix: Crate version 2.0.11 already published on crates.io

**Issue #176** | CLOSED | Created: 2026-07-15 | Updated: 2026-07-15

## Issue
The Release Strict workflow (run #79546726451) failed with the following error:

```
error: failed to publish conxius-enclave-sdk v2.0.11 to registry at https://crates.io

Caused by:
  the remote server responded with an error (status 400 Bad Request): crate version `2.0.11` is already uploaded
```

**Root Cause:** Version `2.0.11` is already published on crates.io, preventing the release workflow from completing successfully.

## Solution
The version number in `Cargo.toml` needs to be incremented before attempting to publish again.

### Steps to Fix:
1. Update version in `Cargo.toml` from `2.0.11` to `2.0.12` (or appropriate semantic version), make sure all versioning across repo as required
2. Commit the version bump: `git commit -m "chore: bump version to 2.0.12"`
3. Push changes and re-trigger the release workflow

- Failed job: https://github.com/Conxian/conxius-enclave-sdk/actions/runs/29387278023/job/87263487246
- Workflow: Release Strict (`.github/workflows/release-strict.yml`)
