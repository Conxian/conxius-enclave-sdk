# Fix: Invalid Handlebars syntax in attest-build-provenance subject parameter

**Issue #169** | CLOSED | Created: 2026-07-13 | Updated: 2026-07-13

## Problem
The provenance attestation workflow uses invalid Handlebars template syntax in the `subject` parameter of the `actions/attest-build-provenance` action.

**Current (broken):**
```yaml
- name: Generate SLSA Provenance
  uses: actions/attest-build-provenance@...
  with:
    subject: "dist/{{ checksum }}"
    artifact_path: dist/checksums.txt
    sha256: ${{ steps.checksum.outputs.checksum }}
```

## Root Cause
The `subject` parameter uses Handlebars syntax `{{ checksum }}` which is not a valid GitHub Actions variable reference. GitHub Actions uses `${{ }}` syntax, not Handlebars `{{ }}`.

## Solution
Replace the Handlebars syntax with proper GitHub Actions syntax or use `subject-path`:

**Option 1: Use subject-path (recommended)**
```yaml
- name: Generate SLSA Provenance
  uses: actions/attest-build-provenance@v1
  with:
    subject-path: dist/libconclave_sdk.so
```

**Option 2: Use GitHub Actions expression syntax**
```yaml
- name: Generate SLSA Provenance
  uses: actions/attest-build-provenance@v1
  with:
    subject: "dist/libconclave_sdk.so"
    sha256: ${{ steps.checksum.outputs.checksum }}
```
