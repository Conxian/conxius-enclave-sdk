# [P0] Replace or quarantine threshold and settlement protocol placeholders

**Issue #197** | OPEN | Created: 2026-07-20 | Updated: 2026-07-20

The audit records placeholder or simulated FROST, Fedimint, Ark, and BitVM2 behavior as P0 high-impact protocol gaps. Until each path is either protocol-conformant and independently reviewed or explicitly unsupported and fail closed, it must not be treated as a production threshold-signing or settlement capability.

Refs #191

## Scope
- Replace or quarantine the FROST, Fedimint, Ark, and BitVM2 paths in `src/protocol/frost.rs`, `src/protocol/nexus/fedimint.rs`, `src/protocol/ark.rs`, and `src/protocol/bitvm2.rs`.
- Separate documented interfaces and development simulations from production protocol behavior.
- Add typed unsupported or disabled outcomes wherever an audited implementation is not available.
- Ensure protocol paths do not expose secret material or rely on structural hashes as a substitute for cryptographic evidence.

## Evidence
- [Audit PR #193](https://github.com/Conxian/conxius-enclave-sdk/pull/193) identifies placeholder and simulated high-impact protocol behavior as a P0 finding.
- Historical issues #173, #174, and #180, together with merged [PR #190](https://github.com/Conxian/conxius-enclave-sdk/pull/190) and [PR #192](https://github.com/Conxian/conxius-enclave-sdk/pull/192), provide related integration or documentation context but do not replace audited implementations or production disablement.
- Relevant implementation paths: `src/protocol/frost.rs`, `src/protocol/nexus/fedimint.rs`, `src/protocol/ark.rs`, and `src/protocol/bitvm2.rs`.

## Acceptance criteria
- Each protocol has either an audited, protocol-conformant implementation with reproducible vectors and integration evidence, or a typed unsupported/disabled production path.
- Production code cannot silently select a structural, simulated, or placeholder implementation for threshold signing, settlement, challenge, or recovery decisions.
- Protocol-specific official or independently checked vectors cover happy paths, invalid proofs, malformed messages, replay, duplicate participation, and boundary conditions.
- Integration tests demonstrate fail-closed behavior when required protocol or hardware evidence is absent.
- Secret material never leaves its approved boundary through logs, telemetry, errors, test fixtures, or public interfaces.

## Dependencies
- Depends on the hardware-backed signing gate in #195 and canonical cryptographic verification in #196.
- Coordinates with the asset/address fail-closed work in this backlog; the final acceptance gate must not include an unsupported settlement capability as production-ready.
